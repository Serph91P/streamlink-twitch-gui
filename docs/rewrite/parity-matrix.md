# Legacy parity matrix

This matrix inventories the user-visible NW.js and Ember behavior from source. "Retained" means the rewrite currently provides the same user goal, not necessarily the same route or presentation. "Redesigned" means the goal remains with a different security or domain model. "Deprecated" means the underlying integration is obsolete. "Intentionally removed" means the rewrite does not carry the behavior forward.

## Storage and migration boundary

| Legacy namespace  | Contents                               | Disposition           | Rationale and source                                                                                                                                                                                                                                                  |
| ----------------- | -------------------------------------- | --------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `settings`        | Singleton global settings record       | Redesigned            | Supported non-secret values are previewed and explicitly imported into schema v1. Unknown values stay unsupported. Adapter: `src/app/data/models/settings/adapter.js:1-6`; model: `src/app/data/models/settings/model.js:11-18`.                                      |
| `channelsettings` | Sparse records keyed by Twitch user ID | Redesigned            | Every channel override is shown in preview, but remains unsupported until a typed channel model exists. No speculative schema is added. Adapter: `src/app/data/models/channel-settings/adapter.js:1-6`; model: `src/app/data/models/channel-settings/model.js:11-35`. |
| `auth`            | OAuth token, scope, and date           | Intentionally removed | Plaintext OAuth is flagged as sensitive and never imported. The rewrite uses device flow and OS credential storage. `src/app/data/models/auth/model.js:7-26`; `src/app/services/auth/service.js:248-277`.                                                             |
| `search`          | Search query, filter, and date history | Redesigned            | Search remains, while rewrite UI history is local non-secret state and is not imported. `src/app/data/models/search/model.js:6-21`; `next/src/state/settings.ts:8-40`.                                                                                                |
| `window`          | Position, size, and maximized state    | Intentionally removed | The Tauri shell currently relies on platform window defaults. `src/app/data/models/window/model.js:5-13`.                                                                                                                                                             |
| `versioncheck`    | Last version and next-check metadata   | Intentionally removed | Operational NW.js update metadata is not a user setting and is incompatible with the release updater. `src/app/data/models/versioncheck/model.js:5-11`.                                                                                                               |
| `app`             | Pre-split aggregate of all namespaces  | Redesigned            | The read-only importer recognizes this old envelope without rewriting or deleting it. Split behavior: `src/app/init/initializers/localstorage/namespaces.js:1-16`.                                                                                                    |

The legacy adapter stores each namespace in `nw.Window.window.localStorage` (`src/app/init/initializers/localstorage/localstorage.js:1-4`), not in the Tauri webview origin. The package name is `streamlink-twitch-gui` (`package.json:2`), so NW.js 0.83 documents its `App.dataPath` as `%LOCALAPPDATA%/streamlink-twitch-gui` on Windows, `~/.config/streamlink-twitch-gui` on Linux, and `~/Library/Application Support/streamlink-twitch-gui/Default` on macOS. Current Chromium-backed releases keep localStorage below the active profile in `Local Storage/leveldb`; older releases can contain SQLite-era `.localstorage` files. Neither format is a stable application interchange API, and opening a live LevelDB directly risks lock and consistency failures.

The rewrite therefore does not claim automatic discovery. It accepts only an explicit user-selected [legacy namespace JSON export](legacy-settings-import.md), passes that object to the Rust preview/import commands, and never reads the new webview's localStorage. The old upgrade initializer has no schema versions (`src/app/init/initializers/localstorage/initializer.js:12-44`), so migration treats malformed, partial, and future values independently. Original profile data and the selected export file are never mutated.

## Routes and user journeys

Legacy routes are declared in `src/app/router.js:4-38`. Rewrite routes are stateful, keyboard-accessible destinations in `next/src/components/layout/AppLayout.tsx:3-53` and `next/src/routes/BrowseRoute.tsx:344-369`.

| Legacy route                                                      | Disposition           | Rewrite behavior and rationale                                                                                                                                            |
| ----------------------------------------------------------------- | --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `/`, `/streams`                                                   | Retained              | Live stream browsing is the default `Live` destination.                                                                                                                   |
| `/games`, `/games/:game_id`                                       | Redesigned            | Top categories are retained; category cards replace nested Ember game routes. Category-specific stream listing is not yet retained.                                       |
| `/search?filter&query`                                            | Retained              | Combined channel and category search is retained without encoding transient state into a router URL.                                                                      |
| `/channel/:user_id`                                               | Retained              | Search opens live or offline channel detail and playback controls.                                                                                                        |
| `/channel/:user_id/teams`                                         | Intentionally removed | Twitch teams are not part of the supported typed Helix surface.                                                                                                           |
| `/channel/:user_id/settings`                                      | Redesigned            | Per-channel values are represented in migration preview but have no mutable rewrite target yet.                                                                           |
| `/watching`                                                       | Redesigned            | The rewrite owns one active playback process and presents its status in channel detail instead of a multi-stream watching route.                                          |
| `/user`, `/user/auth`                                             | Redesigned            | OAuth implicit login and auth routes are replaced by device authorization and OS credential storage. Backend support exists; a dedicated frontend login route is pending. |
| `/user/followed-streams`                                          | Retained              | `Following` loads authenticated followed live streams.                                                                                                                    |
| `/user/followed-channels`                                         | Retained              | `Channels` loads authenticated followed channels.                                                                                                                         |
| `/team/:team_id`, `/team/:team_id/members`, `/team/:team_id/info` | Intentionally removed | Twitch team browsing is outside the typed Helix contract.                                                                                                                 |
| `/settings`, `/settings/main`                                     | Retained              | One settings control room replaces nested Ember settings routes.                                                                                                          |
| `/settings/streams`, `/settings/streaming`, `/settings/player`    | Redesigned            | Supported playback values are grouped around typed Streamlink, player, quality, and codec controls.                                                                       |
| `/settings/chat`                                                  | Intentionally removed | Arbitrary executable chat launchers and token substitution violate the narrow command boundary.                                                                           |
| `/settings/gui`                                                   | Redesigned            | Theme and language remain; NW.js-specific window and taskbar policies are removed.                                                                                        |
| `/settings/lists`                                                 | Intentionally removed | This declared legacy route has no corresponding source template or route implementation.                                                                                  |
| `/settings/languages`                                             | Redesigned            | A typed locale setting replaces the language-filter submenu. Stream-language filtering is not retained.                                                                   |
| `/settings/hotkeys`                                               | Redesigned            | One validated global play-or-stop accelerator replaces renderer action bindings.                                                                                          |
| `/settings/notifications`                                         | Redesigned            | Native live-channel and playback-error intents replace provider and click-action configuration.                                                                           |
| `/settings/channels`                                              | Redesigned            | Existing channel values are visible during migration; editing awaits a typed channel model.                                                                               |
| `/about`                                                          | Intentionally removed | Build metadata does not currently require a dedicated route.                                                                                                              |

| Legacy field                    | Disposition           | Rationale                                                                                                    |
| ------------------------------- | --------------------- | ------------------------------------------------------------------------------------------------------------ |
| `streaming_quality`             | Redesigned            | A future typed override can reuse semantic quality constraints; fixed IDs are not imported now.              |
| `streaming_low_latency`         | Redesigned            | The value is preserved only in preview until a typed playback override exists.                               |
| `streaming_twitch_extra_codecs` | Deprecated            | Legacy persisted this field but playback did not apply it (`src/app/services/streaming/service.js:263-276`). |

## Retained critical-path coverage

Playwright covers the production frontend entry at desktop and narrow widths while replacing only `window.__TAURI_INTERNALS__.invoke`. Journeys cover live browse, categories, authenticated followed lists, search, channel detail, dynamic inspection, launch, quality relaunch, stop, settings persistence, and preview/confirmed legacy import. Backend and Rust tests cover device authentication, logout, tray and hotkey runtime behavior, notification dispatch, process recovery, and migration edge cases that cannot be meaningfully exercised in a browser-only runner.
